# Unified Chat — Twitch + Kick

صفحة تعرض شات تويتش وكيك مدموجين في خلاصة واحدة، كل رسالة موسومة بأيقونة
المنصة (TW / KK). اكتب اسم المستخدم فقط — بدون تسجيل دخول أو كلمة مرور.

## التشغيل

```bash
npm install
npm start
```

ثم افتح: `http://localhost:3000`

## البنية (كيف يبقى الاتصال حياً على الجوال)

المتصفح على الجوال يُجمَّد ويقطع اتصالاته (WebSockets) عند الخروج للتطبيقات
الأخرى — هذا سلوك نظام التشغيل ولا يمكن منعه من داخل المتصفح. لذلك المنطق
مقسّم كما يلي:

- **`server.js` (السيرفر، Node):** هو من يتصل بتويتش وكيك ويبقي الاتصالات
  مفتوحة 24/7 (الـ Node على كمبيوتر/سيرفر لا يتجمد مثل الجوال). يخزّن آخر
  **200 رسالة** لكل جلسة، ويعرض لكل عميل جديد/معيد اتصال هذه الرسائل
  (snapshot) فلا تضيع أي رسالة وقت غيابك. كما يُعيد الاتصال تلقائياً بالمنصتين
  عند أي قطع. وعند مغادرة آخر عميل لا تُدمَّر الجلسة فوراً، بل تُمنح فترة سماح
  (30 دقيقة) تبقى حية وخلالها تلتقط الرسائل؛ لو رجع أي عميل تُلغى الفترة، وإلا
  تُدمَّر تلقائياً مع إغلاق الاتصالات بالمنصتين.
- **`public/app.js` (المتصفح):** يتصل بسيرفرك فقط عبر WebSocket واحد على
  `/ws`. عند الرجوع للصفحة من الخلفية (`visibilitychange`) أو عند القطع،
  يُعيد الاتصال تلقائياً (تراجع أسي حتى 10 ثوانٍ)، ثم يستقبل الـ snapshot
  فيستعيد الرسائل التي فاتته.

كل جلسة (مزيج اسم تويتش + اسم كيك) مستقلة بعملائها وخلاصتها، وتبقى حية لمدة
30 دقيقة بعد مغادرة آخر عميل قبل أن تُدمَّر تلقائياً.

## كيف تعمل كل منصة

- **Twitch:** يتصل السيرفر بخادم المحادثة العام عبر WebSocket
  (`wss://irc-ws.chat.twitch.tv`) كمشاهد مجهول للقراءة فقط (بدون دخول أو مفتاح
  API). هذه الطريقة القياسية التي تستخدمها إضافات عرض الشات الخارجية.
- **Kick:** لا يوجد API رسمي للقراءة بدون دخول، لذلك يتصل السيرفر بنفس
  WebSocket (Pusher) الذي يستخدمه المتصفح على kick.com. غير رسمي وقد يتوقف
  إذا غيّرت كيك شيئاً من طرفها.

### إذا توقف شات كيك

غالباً غيّرت كيك «app key» الخاص بـ Pusher. للإصلاح:
1. افتح kick.com على أي قناة مباشرة.
2. افتح DevTools ← تبويب Network ← رشّح بـ `pusher`.
3. سترى URL مثل `wss://ws-us2.pusher.com/app/XXXXXXX?...`.
4. انسخ القيمة بعد `/app/` واستبدل `KICK_PUSHER_APP_KEY` في `server.js`.

## ملاحظات

- يكفي إدخال اسم واحد (تويتش أو كيك) وتعمل الصفحة بخلاصة واحدة.
- إذا لم يكن الحساب قد بثّ مسبقاً أو غير موجود، ستظهر رسالة خطأ واضحة في الخلاصة
  بدل الفشل الصامت.
- البروكسي `server.js` صار الآن أيضاً طبقة الربط الفعلية بالمنصتين والمسؤول عن
  إبقاء الاتصال حياً وتخزين الرسائل وإعادة الاتصال.
