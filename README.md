# Unified Chat — Twitch + Kick + TikTok + YouTube

صفحة تعرض شات تويتش وكيك وتيكتوك ويوتيوب مدموجين في خلاصة واحدة، كل رسالة
موسومة بأيقونة المنصة (TW / KK / TT / YT). اكتب اسم المستخدم فقط — بدون
تسجيل دخول أو كلمة مرور.

## التشغيل

```bash
npm install
npm start
```

ثم افتح: `http://localhost:3000`

## الإعدادات عبر ملف `.env`

كل الأسرار والإعدادات تُدار من ملف `.env` في جذر المشروع (يُتجاهَل في git).
هذا أفضل من تمريرها في سطر الأوامر حيث تظهر في قائمة العمليات (`ps`/Task Manager).

```bash
cp .env.example .env      # ثم حرّر ‎.env واملأ القيم
npm start
```

قالب ‎`.env.example` يشرح كل متغيّر. كل القيم اختيارية؛ عند تركها فارغة يعمل
الموقع مفتوحاً (للتطوير المحلي).

```env
SITE_PASSWORD=              # كلمة مرور الموقع (افرغها = موقع مفتوح)
TWITCH_CLIENT_ID=           # OAuth لكتابة الشات باسمك
TWITCH_CLIENT_SECRET=
TWITCH_REDIRECT_URI=        # يُبنى تلقائياً إن تُرك فارغاً
TIKTOK_SIGN_API_KEY=        # مفتاح Euler Stream لتيكتوك
PORT=3000
```

> لا يزال بالإمكان تمرير أي متغيّر في سطر الأوامر أيضاً (`SITE_PASSWORD=x npm start`)
> وستلغي قيمته ما في ‎`.env`. لكن ‎`.env` أبسط وأأمن للإنتاج.

### حماية الموقع بكلمة مرور (مستحسن للنشر العام)

اضبط `SITE_PASSWORD` في ‎`.env`.

- تُطلب كلمة المرور في شاشة الدخول الأولى قبل الوصول لأي شيء.
- تُتحقّق على مستوى **HTTP** (`/api/auth`) و**WebSocket** (`/ws`): لا يمكن لأي
  عميل الانضمام للشات أو إرسال رسالة قبل اجتيازها.
- تُحفظ في المتصفح (`localStorage`) بعد أول إدخال ناجح حتى لا تُعاد كل مرة.
- لو لم تُضبط (`SITE_PASSWORD` فارغة)، يبقى الموقع مفتوحاً تماماً.

### مشغل فيديو تويتش (اختياري)

يعمل فوراً بدون إعداد: بعد اختيار قناة تويتش يظهر زر **Show Video** أسفل
الشات؛ يبني المشغل في الأعلى (نفس تخطيط تطبيق تويتش على الجوال)، وزر **Hide
Video** يدمّره كاملاً لتوفير الإنترنت والموارد.

> **الاستضافة:** مشغل تويتش يتطلّب أن يكون نطاق موقعك مسجّلاً كـ **Embed
> domain** على تويتش. للنطاقات المخصّصة سجّلها في
> https://dashboard.twitch.tv/settings → Embed. `localhost` يعمل تلقائياً للتطوير.

### ربط حساب تويتش للكتابة باسمك (اختياري)

القراءة تعمل مجهولاً بلا إعداد. أما الكتابة باسم حسابك الشخصي فتتطلب تسجيل
تطبيق على Twitch:

1. اذهب إلى https://dev.twitch.tv/console وأنشئ تطبيقاً.
2. انسخ **Client ID** وأنشئ **Client Secret**.
3. في إعدادات التطبيق، أضف **OAuth Redirect URL**:
   - محلياً: `http://localhost:3000/auth/twitch/callback`
   - للنشر: `https://نطاقك.com/auth/twitch/callback`
4. ضع القيم في ‎`.env`:

```env
TWITCH_CLIENT_ID=your_id
TWITCH_CLIENT_SECRET=your_secret
TWITCH_REDIRECT_URI=http://localhost:3000/auth/twitch/callback
```

> `TWITCH_REDIRECT_URI` اختياري؛ لو تركته فارغاً يُبنى تلقائياً من نطاق الطلب.
> لكن خلف بروكسي/HTTPS يُفضّل ضبطه يدوياً ليطابق ما سجّلته في Twitch Console.

عندئذٍ يظهر زر **Connect Twitch to chat** أسفل الشات. الضغط عليه يفتح صفحة
تفويض تويتش لمرة واحدة، ثم يُحفظ الـ Token في المتصفح بشكل دائم؛ في الزيارات
التالية تدخل بحسابك تلقائياً دون إعادة تفويض. من لم يربط حسابه يبقى مجهولاً
للقراءة فقط.

## البنية (كيف يبقى الاتصال حياً على الجوال)

المتصفح على الجوال يُجمَّد ويقطع اتصالاته (WebSockets) عند الخروج للتطبيقات
الأخرى — هذا سلوك نظام التشغيل ولا يمكن منعه من داخل المتصفح. لذلك المنطق
مقسّم كما يلي:

- **`server.js` (السيرفر، Node):** هو من يتصل بتويتش وكيك ويبقي الاتصالات
  مفتوحة 24/7 (الـ Node على كمبيوتر/سيرفر لا يتجمد مثل الجوال). يخزّن آخر
  **200 رسالة** لكل جلسة، ويعرض لكل عميل جديد/معيد اتصال هذه الرسائل
  (snapshot) فلا تضيع أي رسالة وقت غيابك. كما يُعيد الاتصال تلقائياً بالمنصتين
  عند أي قطع. وعند مغادرة آخر عميل لا تُدمَّر الجلسة فوراً، بل تُمنح فترة سماح
  (30 دقيقة) تبقى حية وخلالها تلتقط الرسائل؛ لو رجع أي عميل تُلغى الفترة، وإلا
  تُدمَّر تلقائياً مع إغلاق الاتصالات بالمنصتين.
- **`public/app.js` (المتصفح):** يتصل بسيرفرك فقط عبر WebSocket واحد على
  `/ws`. عند الرجوع للصفحة من الخلفية (`visibilitychange`) أو عند القطع،
  يُعيد الاتصال تلقائياً (تراجع أسي حتى 10 ثوانٍ)، ثم يستقبل الـ snapshot
  فيستعيد الرسائل التي فاتته.

كل جلسة (مزيج اسم تويتش + اسم كيك + اسم تيكتوك) مستقلة بعملائها وخلاصتها،
وتبقى حية لمدة 30 دقيقة بعد مغادرة آخر عميل قبل أن تُدمَّر تلقائياً.

## كيف تعمل كل منصة

- **Twitch:** يتصل السيرفر بخادم المحادثة العام عبر WebSocket
  (`wss://irc-ws.chat.twitch.tv`) كمشاهد مجهول للقراءة فقط (بدون دخول أو مفتاح
  API). هذه الطريقة القياسية التي تستخدمها إضافات عرض الشات الخارجية.
- **Kick:** لا يوجد API رسمي للقراءة بدون دخول، لذلك يتصل السيرفر بنفس
  WebSocket (Pusher) الذي يستخدمه المتصفح على kick.com. غير رسمي وقد يتوقف
  إذا غيّرت كيك شيئاً من طرفها.
- **TikTok:** لا يوجد API رسمي للقراءة. السيرفر يستخدم مكتبة
  `tiktok-live-connector` التي تتصل بخدمة Webcast الداخلية لتيكتوك عبر
  WebSocket. لأن تيكتوك يتطلب توقيعاً مشفّراً لكل اتصال، تعتمد المكتبة على
  خدمة **Euler Stream** كـ Sign Server (خطة Community المجانية تكفي للاستخدام
  الشخصي). غير رسمي ويعتمد على طرف ثالث (Euler Stream) للتوقيع.
- **YouTube:** لا يوجد API عملي للقراءة المستمرة (الـ API الرسمي يكلّف 5
  وحدات/قراءة ويتجاوز الـ quota المجاني في ساعات). لذلك السيرفر يستخدم مكتبة
  `youtubei.js` (LuanRT) التي تتصل بـ **InnerTube** — الـ API الداخلي نفسه
  الذي يستخدمه متصفح يوتيوب. لا مفتاح، لا quota، لا حدود. غير رسمي ويعتمد
  على استقرار واجهة InnerTube.

### إذا توقف شات كيك

غالباً غيّرت كيك «app key» الخاص بـ Pusher. للإصلاح:
1. افتح kick.com على أي قناة مباشرة.
2. افتح DevTools ← تبويب Network ← رشّف بـ `pusher`.
3. سترى URL مثل `wss://ws-us2.pusher.com/app/XXXXXXX?...`.
4. انسخ القيمة بعد `/app/` واستبدل `KICK_PUSHER_APP_KEY` في `server.js`.

## تفعيل تيكتوك

تيكتوك يحتاج مفتاح **Euler Stream** لتوقيع الاتصالات (لا يعمل بدونه):

1. أنشئ حساباً مجانياً على [eulerstream.com](https://www.eulerstream.com)
   (خطة Community: 2,500 طلب/يوم، تكفي للاستخدام الشخصي).
2. انسخ مفتاح الـ API من لوحة التحكم.
3. شغّل السيرفر مع متغيّر البيئة:
   ```bash
   TIKTOK_SIGN_API_KEY="مفتاحك_هنا" npm start
   ```
   أو على ويندوز (PowerShell):
   ```powershell
   $env:TIKTOK_SIGN_API_KEY="مفتاحك_هنا"; npm start
   ```

بدون المفتاح، تظهر حالة تيكتوك `error` مع رسالة توضّح أن المفتاح مطلوب، بينما
تستمر تويتش وكيك بالعمل بشكل طبيعي.

> **ملاحظة:** تيكتوك يعرض فقط الشات أثناء البث المباشر. لو لم يكن الحساب
> مباشراً الآن، تظهر رسالة "غير مباشر" بدل الفشل الصامت.

## يوتيوب

يوتيوب لا يحتاج أي إعداد — يعمل فوراً بدون مفتاح. الفرق الوحيد أن الإدخال
**رابط البث المباشر** أو **معرّف الفيديو** (وليس username):

- `https://www.youtube.com/watch?v=XXXXXXXXXXX` ✅
- `https://youtu.be/XXXXXXXXXXX` ✅
- `https://www.youtube.com/live/XXXXXXXXXXX` ✅
- `XXXXXXXXXXX` (معرّف الفيديو فقط، 11 حرفاً) ✅

> **ملاحظات:**
> - يوتيوب يربط الشات بـ **بث محدّد**، لا بحساب؛ لذلك يجب أن يكون البث
>   **مباشراً الآن**. لو انتهى، تظهر رسالة «انتهى بث يوتيوب».
> - المكتبة تُعيد الاتصال تلقائياً داخلياً عند الأخطاء العابرة (حتى 10
>   مرات). عند انتهاء البث فعلاً تتوقف ولا تُعيد المحاولة بلا فائدة.

## ملاحظات

- يكفي إدخال اسم واحد (تويتش أو كيك أو تيكتوك) وتعمل الصفحة بخلاصة واحدة.
- إذا لم يكن الحساب قد بثّ مسبقاً أو غير موجود، ستظهر رسالة خطأ واضحة في الخلاصة
  بدل الفشل الصامت.
- البروكسي `server.js` صار الآن أيضاً طبقة الربط الفعلية بالمنصتين والمسؤول عن
  إبقاء الاتصال حياً وتخزين الرسائل وإعادة الاتصال.
